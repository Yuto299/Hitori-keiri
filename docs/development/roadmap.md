## 0. 進め方の原則

- **動く順に積む**:常に「起動して触れる」状態を保ちながら縦に機能を通す。
- **ロジックはテスト優先**:CSV・プラン判定・枚数・店名正規化は単体テストでガード。
- **仕様未確定は止める**:要件にない判断が要るときはオーナーに確認(勝手に決めない)。
- **変更はドキュメントに残す**:構成・方針を変えたら本ファイルか docs/development を更新。
- **課金・外部公開はオーナー確認**:EAS Build/Submit、ストア申請、Supabase/Anthropic の課金変更など。

## 3.5 動かし方(現状)

```bash
npm install          # 初回
npm run web          # ブラウザで http://localhost:8081 を開く(動作確認はこれが最短)
# iOS/Android 実機は Development Build が必要(SDK55はExpo Go非対応)
```

**Webで触れること(現状)**: ホーム → 「レシートを撮る」→ 画像選択 → OCR(モック)→ 確認・編集 → 保存 → レシート一覧 → 詳細/削除、CSV出力(汎用は全プラン/各社はLight以上)、設定でプラン切替(機能ゲートの体験)。
※ Web ではレシートはインメモリ保持(リロードで消える)。OCR はモック応答。画像保存は Light/Pro 選択時のみ。これらは設計どおりの暫定で、ネイティブ/フェーズ4以降で本実装。
