## 0. 進め方の原則

- **動く順に積む**:常に「起動して触れる」状態を保ちながら縦に機能を通す。
- **ロジックはテスト優先**:CSV・プラン判定・枚数・店名正規化は単体テストでガード。
- **仕様未確定は止める**:要件にない判断が要るときはオーナーに確認(勝手に決めない)。
- **変更はドキュメントに残す**:構成・方針を変えたら本ファイルか docs/development を更新。
- **課金・外部公開はオーナー確認**:EAS Build/Submit、ストア申請、Supabase/Anthropic の課金変更など。

## 2. フェーズ詳細

### フェーズ1:ドメイン基盤・ナビゲーション(現在地)
**ゴール**:アプリが起動し、ドメイン型とプランロジックがテストで固まっている。

- [ ] ドメイン型の定義(`shared/types/receipt.ts` 等):Receipt / ReceiptMemo / OcrExtraction
- [ ] 勘定科目マスタ(`constants/categories.ts`)
- [ ] プラン判定ロジック(`features/billing/`):機能ゲート(FR-21)・枚数(FR-22)+ テスト
- [ ] ナビ:4タブ(ホーム/レシート/出力/設定)の骨組み。各画面プレースホルダ
- [ ] ローカルDB初期化(`lib/db/`:expo-sqlite スキーマ)
- 検証:`npm test` / `npm run typecheck` / 起動確認

### フェーズ2:撮影 → 確認 → 保存(OCRはモック)
**ゴール**:レシートを撮って、内容を確認・編集して、端末に保存できる(OCRはダミー応答)。

- [ ] カメラ(`features/capture/`):撮影(FR-01)・画像取込(FR-02)・連続撮影(FR-03)
- [ ] OCR サービスの抽象インターフェース + **モック実装**(後でClaudeに差し替え)
- [ ] 確認画面(`features/receipt-review/`):承認(FR-07)・修正(FR-08)・メモ(FR-09)
- [ ] レシート一覧・詳細(`features/receipts/`):閲覧(FR-11)・編集削除(FR-14)
- 検証:撮る→確認→保存→一覧表示 のコアフローが手で通る

### フェーズ3:Supabase 連携(認証・同期)
**ゴール**:ログインでき、データがサーバに同期され、機種変更で復元できる。

- [ ] Supabase プロジェクト作成(※オーナー確認のうえ。無料枠)
- [ ] 認証(`features/auth/`):Apple / Google / メール(FR-25)
- [ ] RLS ポリシー(`supabase/migrations/`):user_id 単位の分離
- [ ] ローカル(SQLite)↔ リモート(Postgres)同期。購入復元の土台
- 検証:ログイン→撮影→別端末/再インストールで復元

### フェーズ4:OCR 本実装(Claude API)
**ゴール**:実レシートから日付/金額/店名/科目が自動抽出される。

- [ ] Edge Function `ocr-receipt`:Claude API(Vision + Structured Outputs)※キーはサーバ
- [ ] クライアント前処理:画像リサイズ([第6章 6.3.1](../requirements/06-ocr-ai.md#631-入力画像))
- [ ] confidence による低確度ハイライト(確認画面)
- [ ] 画像保存ポリシー(FR-12):Free即削除 / Light30日 / Pro無期限
- 検証:実レシート数種で抽出精度・コストを実測(→ 第8章へ反映)

### フェーズ5:CSV 出力
**ゴール**:汎用CSV → 各社形式で書き出し、共有できる。

- [ ] 汎用CSV(`features/export/formatters/generic.ts`)+ テスト
- [ ] freee / マネフォ / 弥生(各社最新仕様で列確定 → [第5章](../requirements/05-data-model-csv.md#553-会計ソフト形式lightpro--fr-1618))
- [ ] 期間指定(FR-19)・OS共有シート連携
- 検証:各形式を実ファイルで出し、サンプル取込を確認

### フェーズ6:課金・プラン
**ゴール**:プラン購入で機能が解放され、課金壁が文脈どおり出る。

- [ ] 課金基盤(RevenueCat 採用是非を検証 → [tech-stack 6章](./tech-stack.md#6-未確定要検証後続タスク))※オーナー確認
- [ ] 機能ゲート結線(FR-21)・枚数上限UI(FR-22)
- [ ] アップグレード画面 S-07・課金壁の発火([第4章 4.6](../requirements/04-screen-design.md#46-課金壁の発火マップまとめ))
- 検証:Free/Light/Pro で機能の出し分け、購入・復元

### フェーズ7:Pro機能・体験の仕上げ
- [ ] レシート検索(FR-13)/ 音声メモ(FR-10)/ AI学習(FR-06)
- [ ] オンボーディング S-01(FR-24)/ Free削除バナー(FR-26)
- [ ] デザインA トーンの作り込み(4タブ化・アイコン・余白)

### フェーズ8:非機能の仕上げ
- [ ] オフライン撮影キュー([第7章 7.3](../requirements/07-non-functional.md#73-オフライン同期))
- [ ] エラー監視(Sentry)・退会/データ削除導線
- [ ] ストア準備(プライバシー開示・IAP)※申請はオーナー確認
