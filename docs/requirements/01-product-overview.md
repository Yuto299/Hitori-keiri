## 1.1 プロダクト名とコンセプト

| 項目 | 内容 |

## 1.2 背景:なぜ今このプロダクトが必要か

日本のフリーランス・副業従事者は増加傾向にある一方、**多くの個人事業主は「会計」を年1回の確定申告前に集中処理している**実態がある。日々コツコツ記帳する習慣を持たず、年明けに「靴箱いっぱいのレシート」と向き合うことになる層が大多数を占める。

既存の会計SaaS(freee・マネーフォワード・弥生)は**「日々記帳する前提」の総合会計ソフト**として設計されており、以下のミスマッチが存在する。

- **機能過剰**:仕訳・請求書・給与計算など、副業勢には不要な機能が標準装備
- **価格負担**:freee会計スターター¥1,078/月〜、年¥12,936の固定費は副業初期層には重い
- **学習コスト**:勘定科目・仕訳の概念を理解しないと使いこなせない

## 1.3 解くべき課題(コアペイン)

> **「確定申告前にレシートの山を見て絶望する」**

MVPはこの**年1回の集中作業の痛み**にフォーカスする。日々の記帳習慣化や複式簿記の理解は要求しない。レシートさえスマホで撮っておけば、申告期に会計ソフトへ取り込めるCSVが手に入る状態をゴールとする。

## 1.4 ソリューション

3タップで完結する「撮るだけ経理」体験を提供する。

1. **撮る**:レシートをカメラで撮影
2. **確認**:OCRが抽出した日付・金額・店名・勘定科目候補をワンタップで承認
3. **出力**:確定申告期にfreee/マネフォ/弥生形式のCSVを書き出し、ユーザー自身が会計ソフトへインポート

本アプリは**会計ソフトに接続しない**。出口はCSVファイルのみ。これにより、

- 会計ソフトのAPI仕様変更リスクを回避
- 「会計ソフトを使っていない人」も顧客になる(Excel運用の自営業者、確定申告を税理士に丸投げしている層)
- 開発スコープを最小化し、1ヶ月でMVPリリース可能

## 1.5 差別化:なぜ勝てるか

| 軸 | freee会計スターター | ひとり経理 Light |
|---|---|---|
| 月額 | ¥1,078 | **¥480** |
| 年額 | ¥12,936 | **¥4,800**(月額×10相当) |
| スコープ | 総合会計ソフト | **レシート→CSV特化** |
| 操作完了までのタップ数 | 10タップ以上(仕訳画面) | **3タップ** |
| 学習コスト | 簿記知識が前提 | 簿記知識不要(科目は自動候補) |
| ターゲット心理 | 「ちゃんと経理する人」 | **「申告前だけで済ませたい人」** |

直接の競合機能としては freee「魔法スキャン」や STREAMED があるが、いずれも本体の会計ソフト契約(¥1,000〜)が前提。**「会計ソフトに加入せずレシート機能だけ欲しい」層は市場の空白地帯**であり、ここを取りに行く。

## 1.6 ターゲットセグメント(概要)

詳細ペルソナは[第2章](./02-personas.md)で定義する。本プロダクトは以下2層を同時に狙う。

- **副業・小規模フリーランス**:売上〜300万円 / レシート月20〜50枚 / 確定申告は年1回まとめてやる派 → **Lightプランの主顧客**
- **本業フリーランス初期**:売上300〜800万円 / レシート月50〜150枚 / freeeを試したが使いこなせず解約 → **Proプランの主顧客**

両者の共通項は「**会計ソフトに月¥1,000以上払う価値を感じていない**」こと。

## 1.7 料金プラン

### 1.7.1 プラン構成

| プラン | 月額 | 年額(月額×10相当) | 主な提供価値 |
|---|---:|---:|---|
| **Free** | ¥0 | - | お試し用 / 汎用CSVのみ |
| **Light** | ¥480 | ¥4,800 | 副業・軽めのフリーランス向け |
| **Pro** | ¥780 | ¥7,800 | 本業フリーランス・確定申告対応向け |

年額は月額の2ヶ月分割引。MVPから**月額・年額の両方を提供**する。

### 1.7.2 機能比較

| 機能 | Free | Light | Pro |
|---|:---:|:---:|:---:|
| 月間レシート枚数 | 5枚 | 30枚 | 無制限 |
| 汎用CSV出力 | ○ | ○ | ○ |
| freee形式CSV | ✕ | ○ | ○ |
| マネーフォワード形式CSV | ✕ | ○ | ○ |
| 弥生形式CSV | ✕ | ○ | ○ |
| 画像保存 | **即削除**(OCR後にテキスト化) | 30日間 | 無期限 |
| カテゴリAI学習(店名→科目) | ✕ | ✕ | ○ |
| レシート検索(日付/金額/店名) | ✕ | ✕ | ○ |
| レシート単位のメモ(自由記述/同席者/目的/案件名) | ○ | ○ | ○ |
| 音声入力でメモ記録 | ✕ | ✕ | ○ |

**注1:Free**は撮影画像をOCR処理後ただちに削除する。これによりストレージコストをゼロに抑えつつ、ユーザーには「申告期にCSVを出すまでレシート画像を見返したくなる」=Light/Proへの動機が生まれる。

なお、ユーザーが「写真が消えた!」と誤解しないよう、**Freeプランでは画像削除を事前明示する方針**とする。具体的には、初回起動時のオンボーディング画面で説明 + 撮影画面に常時バナーで明示する。詳細な文言・配置は画面設計フェーズで決定。

**注2:レシートメモ**は全プランで利用可能。レシート1件ごとに以下のフィールドを任意で追加できる。

- **自由記述メモ**(汎用)
- **同席者**(交際費・会議費の必須記録項目。税務調査でも参照される)
- **目的・用途**(「クライアントA との打合せ」など)
- **プロジェクト/案件名**(任意。Phase2でプロジェクト別集計につなげる伏線)

これらメモはCSV出力時の摘要欄に書き出され、会計ソフトで仕訳の補足情報として活用される。**Pro限定で音声入力**に対応し、運転中や外出直後の片手操作でも記録できるようにする。

### 1.7.3 課金動機(コンバージョン設計)

| 移行 | 課金が発生する瞬間 |
|---|---|
| **Free → Light** | 確定申告期、freee/マネフォ/弥生にCSVを取り込もうとした瞬間。汎用CSVではインポートできず、対応形式CSVが必要だと気づく |
| **Light → Pro** | 撮ったはずのレシートを過去に遡って探したい瞬間。30日を超えた画像が消えていることに気づき、無期限保存と検索機能を求める |

この2段階の課金壁は **「無料で使い始めて、必要になった時だけ払う」** という副業層の心理に沿う設計。とくにLight→Proの動機は**確定申告期に最大化**するため、年額プランの解約率を下げる効果が期待できる。

### 1.7.4 「カテゴリAI学習」の実装方針(参考)

UIラベルは "AIが学習" と表現するが、**実装はルールベース**で開始する。

- 店名(OCR抽出した加盟店名)と勘定科目の対応をユーザー単位で記録
- 次回以降、同じ店名のレシートに過去採用率の高い科目を**第一候補として提示**
- ML推論や類似店名のクラスタリングはPhase2以降

これにより1ヶ月MVPの開発工数を圧縮しつつ、ユーザー体験上は十分「賢く」感じられる。

## 1.8 MVPスコープの境界線(やらないことの明示)

開発期間1ヶ月・開発者1名の制約下で、以下は**MVPでは明確に対象外**とする。

- **電子帳簿保存法対応**(タイムスタンプ・訂正削除履歴・検索要件):ユーザー自身による原本保管を前提とする旨をアプリ内で明示。Phase2以降で検討
- **法人向けプラン**(想定¥2,980〜):Phase2以降。MVPは個人事業主に絞る
- **銀行口座/クレジットカード連携**
- **会計ソフトとのAPI直接連携**(CSVのみ)
- **帳簿・確定申告書の作成機能**(複式簿記の仕訳機能を含む)
- **複数人での共有 / チーム機能**
- **請求書発行・売上管理**
- **レシート以外の証憑**(請求書PDF、原本郵送物)の管理

## 1.9 成功指標(初期仮説)

リリース後3ヶ月時点で以下を目標とする([第2章](./02-personas.md)のペルソナ確定後に再調整)。

- DL数:5,000(主にApp Store / Google Play検索流入)
- 月次アクティブ率(月1回以上撮影):30%
- Free → Light 転換率:5〜8%(申告期は跳ね上がる想定)
- Light → Pro 転換率:15%(対Light有料ユーザー / 確定申告期に集中)
- 月次解約率(Light + Pro合算):8%以下
- 年額プラン比率:Light/Pro合計の30%以上

## 1.10 収支シミュレーションの前提(詳細は第8章)

OCRコストはMVPの損益を左右する最大変数のため、本章では前提のみ明記する。

- Freeユーザー:月最大5枚 × OCR単価 が**広告なしでも赤字にならない**水準か要検証
- Lightユーザー:月30枚 × OCR単価 < ¥480 - 決済手数料(約30%) であること
- Proユーザー:無制限としつつ、実利用平均は月100〜150枚と仮置き

OCR API選定(Claude API vs Google Vision API)とともに、第8章で詳細試算する。

## 章末:確定事項

第1章の確認事項は以下のとおり確定した。

1. **「Free画像即削除」のUX文言**:「撮影したレシートはテキスト化後すぐに削除されます。画像を残したい方はLight以上にアップグレードしてください」の方向で確定。ユーザーが「写真が消えた!」と誤解しないよう事前明示する(オンボーディング+撮影画面バナー)。文言の最終調整は画面設計フェーズで行う。

2. **Pro→法人プラン(¥2,980〜)の位置づけ**:**「Pro拡張型」として仮置き**する。すなわち「Pro + DB分離 + 複数ユーザー」の延長線上にある上位プランと仮定する。税理士連携機能を軸にした別系統ではない。詳細はPhase2で確定するが、用語集には予約語として残す。

3. **キャッチコピー**:以下2案を併記で保持し、リリース前のA/Bテスト・リサーチで確定する。

   - B案(ライト路線):「いちばんやさしい、ひとりの経理。」
