## 4.1 本章の方針

[第3章](./03-functional-requirements.md)の機能要件(FR-01〜26)を、具体的な**画面**と**画面遷移(UXフロー)**に割り付ける。

- ビジュアルデザインではなく、**画面の役割・要素・遷移・どのFRを満たすか**を定義する。
- レイアウトは ASCII ワイヤーフレームで示す(色・余白・タイポは画面デザインフェーズで確定)。
- 最優先は「**3タップで終わらせる**」体験([1.4](./01-product-overview.md#14-ソリューション))の実装可能性。

## 4.2 ナビゲーション構造

下部タブ4つ(ホーム/レシート/出力/設定)とする(確定デザイン[4.9](#49-ビジュアルデザイン確定)に準拠)。撮影は中央のタブではなく、**ホーム画面の主役ボタン「レシートを撮る」**として最短動線を確保する。

```
┌─────────────────────────────────────┐
│              (各画面)                │
│                                       │
├─────────────────────────────────────┤
│  [ホーム]  [レシート]  [出力]  [設定]  │   ← 下部ナビ4タブ
└─────────────────────────────────────┘
```

| ナビ項目 | 画面 | 主なFR |
|---|---|---|
| ホーム | ホーム(S-09):今月枚数・最近のレシート・「撮る」 | FR-01, 22 |
| レシート | レシート一覧(S-04) | FR-11, 13, 14 |
| 出力 | CSV出力(S-06) | FR-15〜19 |
| 設定 | 設定・アカウント・プラン(S-08) | FR-20〜23, 25 |

撮影(S-02)はホームの「レシートを撮る」ボタンから起動。アップグレード(S-07)は各所の課金壁から呼び出すモーダルとする。

## 4.4 コアフロー:3タップで終わらせる(撮る→確認→出力)

「3タップ」は**1枚のレシートを記録し終えるまで**の操作数を指す。

```
ホーム
  │  ① [📷 撮る] をタップ        ← タップ1
  ▼
S-02 カメラ撮影
  │  ② シャッターを切る          ← タップ2
  ▼
  (OCR処理中…)
  ▼
S-03 確認・編集
  │  抽出結果(日付/金額/店名/科目)を表示
  │  ③ [承認] をタップ            ← タップ3
  ▼
ホーム(記録完了トースト)
```

- 抽出が正しければ**修正ゼロ**で承認でき、計3タップで完了(FR-07)。
- 修正が必要な項目だけ S-03 上でタップして直す(FR-08)。修正は AI学習(FR-06)に反映。
- CSV出力(S-06)は記録のたびに行うものではなく、**申告期にまとめて**行う別フロー(4.7)。

## 4.5 画面詳細

### S-01 オンボーディング(FR-24, 25)

初回起動時のみ。3〜4枚のスワイプ + 末尾にログイン。

```
┌───────────────────────────┐
│  ① 撮るだけ。3タップで完了    │
│  ② 申告期にCSVで書き出すだけ  │
│  ③ 【重要】Freeは画像をテキスト │
│     化後すぐ削除します        │
│     画像を残すならLight以上へ  │
│  ───────────────────────  │
│  [Appleでサインイン]          │
│  [Googleでサインイン]         │
│  [メールではじめる]           │
└───────────────────────────┘
```

- ③で**Free画像削除を事前明示**([1.7 注1](./01-product-overview.md#172-機能比較)/ FR-24)。開業直後の新人(E)・超ライト層(C)の誤解防止。
- 公開ストア前提でアカウント必須(FR-25)。ログイン手段は 4.8 で詳述。

### S-02 カメラ撮影(FR-01, 02, 03, 26)

```
┌───────────────────────────┐
│ ⚠ Freeプラン:画像は保存され   │ ← FR-26 常時バナー(Freeのみ)
│   ません(テキスト化後に削除)  │
├───────────────────────────┤
│                             │
│      (カメラプレビュー)       │
│                             │
│   今月の残り枚数:あと 3 枚    │ ← Free/Light(FR-22)
├───────────────────────────┤
│  [🖼 取込]   ( ⚪ )   [連続]  │
│   FR-02    シャッター  FR-03  │
└───────────────────────────┘
```

- **FR-26**:Freeのみ、画像削除バナーを撮影画面に常時表示。Light/Proでは非表示。
- **FR-22**:Free/Lightは残り枚数を表示。0枚で[アップグレード](#s-07-アップグレードfr-20-23)へ誘導。
- 連続撮影(FR-03)では撮るごとにサムネイルが積まれ、まとめて S-03 のキューに送る。

### S-03 確認・編集(FR-04〜10)

```
┌───────────────────────────┐
│      (レシート画像サムネ)      │
├───────────────────────────┤
│ 日付  2026-05-25         [✎] │ FR-04/08
│ 金額  ¥1,280             [✎] │
│ 店名  スターバックス      [✎] │
│ 科目  [会議費 ▼] 候補:接待… │ FR-05/06
├───────────────────────────┤
│ メモ                    [🎤] │ FR-09/10(🎤=Pro)
│  同席者 / 目的 / 案件名       │
├───────────────────────────┤
│        [ 承認して保存 ]       │ FR-07
└───────────────────────────┘
```

- **FR-05/06**:科目はドロップダウンで候補提示。Proは過去採用率で第一候補が自動選択(AI学習)。
- **FR-10**:メモの🎤はProのみ活性。Free/Lightではタップ時にアップグレード案内。
- 連続撮影分はこの画面をスワイプで次々承認できる(店舗系自営業 F の大量処理を想定)。

### S-04 レシート一覧(FR-11, 13)

```
┌───────────────────────────┐
│ [🔍 検索(日付/金額/店名)]   │ FR-13(Pro)
├───────────────────────────┤
│ 05/25 スターバックス  ¥1,280 │
│ 05/24 ガソリン       ¥5,400 │ FR-11
│ 05/22 Amazon        ¥3,200 │
│ …                          │
├───────────────────────────┤
│        [ CSVを書き出す ]      │ → S-06
└───────────────────────────┘
```

- **FR-13**:検索バーはProのみ機能。Free/Lightはタップでアップグレード案内(Light→Pro課金壁)。
- 各行タップで S-05 詳細へ。

### S-05 レシート詳細(FR-09, 12, 14)

```
┌───────────────────────────┐
│   (画像 or「画像は削除済み」)  │ FR-12
│   ※Free:テキストのみ表示      │
├───────────────────────────┤
│ 日付 / 金額 / 店名 / 科目     │
│ メモ(同席者・目的・案件名)    │ FR-09
├───────────────────────────┤
│        [編集]    [削除]       │ FR-14
└───────────────────────────┘
```

- **FR-12**:画像の見え方がプランで変わる。Freeは「画像は削除済み(テキストのみ保持)」と明示し、Light→Proへの動機に繋げる。30日経過したLightの画像も同様の表示。

### S-06 CSV出力(FR-15〜19)

```
┌───────────────────────────┐
│ 形式を選択                   │
│  ○ 汎用CSV(全プラン)        │ FR-15
│  ○ freee形式                │ FR-16 ┐
│  ○ マネーフォワード形式       │ FR-17 ├ Light/Pro
│  ○ 弥生形式                  │ FR-18 ┘
├───────────────────────────┤
│ 期間  [2025年(1〜12月) ▼]   │ FR-19
├───────────────────────────┤
│        [ 書き出して共有 ]     │
└───────────────────────────┘
```

- **FR-16〜18**:Freeで対応形式を選ぶと[アップグレード](#s-07-アップグレードfr-20-23)へ(Free→Light課金壁の発火点)。
- 出力後はOS標準の共有シート(メール添付・ファイル保存・クラウド共有)で渡せる。**税理士丸投げ層(D)**はここでCSVをそのまま送付。

### S-07 アップグレード(FR-20, 23)

各課金壁から起動するモーダル。**発火文脈に応じて訴求文を出し分ける**。

```
┌───────────────────────────┐
│  freee形式で書き出すには      │ ← 文脈別の見出し
│  Light以上が必要です          │
├───────────────────────────┤
│   Light ¥480/月 ・ ¥4,800/年 │
│   Pro   ¥780/月 ・ ¥7,800/年 │ FR-20
│   (プラン比較表)             │
├───────────────────────────┤
│   [このプランで続ける]        │
└───────────────────────────┘
```

- 発火点(FR-23):①Free→Light=対応形式CSV選択時/枚数上限到達時 ②Light→Pro=30日超画像の閲覧/検索/音声入力タップ時。
- 月額/年額を選択して購入(FR-20)。課金は App Store / Google Play のサブスク。

### S-08 設定(FR-21, 22, 25)

- 現在のプランと今月の利用枚数(FR-21, 22)
- アカウント情報・ログアウト・**購入の復元**(FR-25)
- データ同期状態・問い合わせ・規約/プライバシー

## 4.6 課金壁の発火マップ(まとめ)

| 課金壁 | 発火画面 | 操作 | 遷移先 |
|---|---|---|---|
| Free→Light | S-06 CSV出力 | 対応形式を選択 | S-07(freee/マネフォ/弥生訴求) |
| Free→Light | S-02 撮影 | 枚数上限に到達 | S-07(枚数訴求) |
| Light→Pro | S-04 一覧 | 検索バーをタップ | S-07(検索訴求) |
| Light→Pro | S-05 詳細 | 消えた画像を見ようと | S-07(無期限保存訴求) |
| Light→Pro | S-03 確認 | 音声入力🎤をタップ | S-07(音声入力訴求) |

[1.7.3 課金動機](./01-product-overview.md#173-課金動機コンバージョン設計)の設計と一致させる。

## 4.7 申告期フロー(まとめて出力)

日々の「撮る→確認」とは別に、年1回の出力動線を独立させる。

```
S-04 一覧 → [CSVを書き出す] → S-06(形式・期間選択)
  → (Freeなら S-07 課金壁) → 書き出し → OS共有シート → 会計ソフト/税理士へ
```

- これが**Free→Lightの最大の課金タイミング**(申告期に跳ね上がる転換率、[1.9](./01-product-overview.md#19-成功指標初期仮説))。

## 4.8 ログイン手段(FR-25 関連)

公開ストア前提でアカウント必須。MVPの方針(草案):

- **Sign in with Apple**(iOSで他社ソーシャルログインを出す場合、Appleガイドライン上ほぼ必須)
- **Googleでサインイン**
- **メール**(マジックリンク or パスワード)

> 認証基盤・データ同期の具体方式(Firebase Auth / Supabase 等)は第7章(非機能・技術方針)で確定する。本章は「どの画面でログインを出すか」までを定義。

## 4.9 ビジュアルデザイン(確定)

検討した3案(A やさしいミニマル / B 絶望からの解放 / C 暮らしに寄り添う)から、**デザインAをベースに確定**した。確定モックの画像は[デザイン資産](../design/README.md)に保管。

### 4.9.1 採用方針

| 項目 | 確定内容 |
|---|---|
| ベース案 | **デザインA「やさしいミニマル」** |
| テーマ | ライトテーマ(白基調 + グリーンのアクセント) |
| トーン | やさしく・シンプル・初めてでも直感的 |
| ロゴ | 緑のレシートアイコン + 「ひとり経理 / Hitori Keiri」 |
| キャッチコピー | **A案・B案を併用**。主見出しにA案「レシートを撮るだけ。確定申告前の絶望を、3タップで終わらせる。」、補助にB案「いちばんやさしい、ひとりの経理。」 |

> [第1章 1.1](./01-product-overview.md#11-プロダクト名とコンセプト)で「A/Bどちらに振るかはリリース前に確定」としていたが、**両方を主見出し+補助コピーとして併用**する形で確定した。

### 4.9.2 カラーパレット(基準)

| 用途 | 色 | 備考 |
|---|---|---|
| プライマリ(アクセント) | グリーン | 主要ボタン・選択状態・ロゴ |
| 背景 | ホワイト〜淡グレー | カード/リストは白 |
| テキスト | ダークグレー〜黒 | 見出し・本文 |
| 注意・明示バナー | 淡イエロー系 | Free画像削除の明示(S-02/S-05) |

> 正確なカラーコード・タイポグラフィ・余白スケールは、実装時にデザイントークンとして別途定義する。本節は方向性の確定まで。

### 4.9.3 確定デザインでカバーされた画面

採用デザインのモックには以下が含まれ、[4.3 画面一覧](#43-画面一覧)と対応する。

| モック上の画面 | 対応ID | 備考 |
|---|---|---|
| ホーム(今月のレシート枚数・最近のレシート・「レシートを撮る」) | S-04の入口/ホーム | 下部ナビ4項目(ホーム/レシート/出力/設定) |
| レシートを撮影 | S-02 | フラッシュ・シャッター・直近サムネ |
| 内容を確認 | S-03 | 日付/金額/店名/カテゴリ(勘定科目)/メモ |
| レシート一覧 | S-04 | 検索・「すべて/未出力/出力済み」タブ |
| CSVを書き出す | S-06 | freee/マネフォ/弥生/汎用 の選択 |
| 書き出し完了 | S-06後の完了 | ファイル名表示・「ファイルを開く」 |
| プラン | S-08 | 現在プラン・今月の利用状況・画像保存期間 |
| Free画像削除の明示バナー | FR-26 | 「テキスト化後すぐ削除」+「詳しく見る」 |

> 補足:モックの下部ナビは**4項目(ホーム/レシート/出力/設定)**になっており、[4.2](#42-ナビゲーション構造)で草案とした「3タブ+中央撮影ボタン」から**4タブ構成へ更新**された。確定デザインに合わせ、ナビは4タブ(撮影はホームの主役ボタン+各所から起動)とする。

## 章末:確認事項(第4章)

ビジュアルデザインは[4.9](#49-ビジュアルデザイン確定)のとおり**デザインA(やさしいミニマル/グリーン・ライトテーマ)で確定**。ナビは確定デザインに合わせ**4タブ(ホーム/レシート/出力/設定)**に決定した。残る論点は以下。

1. ~~**ナビ構造**~~ → 確定デザインに準拠し4タブで確定([4.2](#42-ナビゲーション構造))。
2. **連続撮影とOCRコスト**:S-03で連続承認できる設計だが、連続撮影は枚数=OCR回数を増やす。Freeの枚数カウント(FR-22)は「撮影枚数」基準でよいか?(撮り直し・失敗分の扱い)
3. **画像が消えた後の詳細画面(S-05)**:Free/期限切れLightで「画像は削除済み(テキストのみ)」と出す方針で違和感ないか?ここがLight/Pro訴求の要なので文言は重要。
4. **ログイン手段**:Apple / Google / メール の3つで想定。MVPで**メールだけ**に絞る等、減らす意向はあるか?

問題なければ第5章(データモデル / CSV仕様)に進みます。
