## 8.1 本章の方針

OCRコストはMVPの損益を左右する最大変数([第1章 1.10](./01-product-overview.md#110-収支シミュレーションの前提詳細は第8章))。本章で**プラン別の単位経済(ユニットエコノミクス)**を試算し、価格設定([1.7](./01-product-overview.md#17-料金プラン))が成立するか検証する。

> 注:数値は**仮置きの前提に基づく試算**であり、実測(OCR精度・実利用枚数・為替・モデル単価)で更新する。意思決定の枠組みを与えるのが目的。

## 8.2 前提(仮置き)

| 変数 | 仮置き値 | 備考 |
|---|---|---|
| OCR単価(1枚) | **¥0.8** | Claude Vision 画像≒1,500トークン+出力。下位モデル想定・保守的上限。為替 ¥155/$ |
| 決済手数料率 | **30%** | App Store / Google Play 標準。小規模は15%もありうる(後述) |
| 為替 | ¥155/$ | 変動リスクあり |
| Free 月間枚数 | 5枚(上限) | [1.7.2](./01-product-overview.md#172-機能比較) |
| Light 月間枚数 | 30枚(上限) | |
| Pro 月間枚数 | 100〜150枚(実利用平均) | 無制限だが実利用の仮置き |

- OCR単価は最重要変数。**¥0.3(楽観)〜¥0.8(保守)** のレンジで感度を見る。
- その他原価:Supabase(DB/Storage/Edge Function)、Sentry等。ユーザー単価に対し小さいと仮定し、本試算では**OCRと決済手数料**を主原価として扱う(固定費は別途8.6)。

## 8.3 プラン別ユニットエコノミクス(月額・1ユーザーあたり)

### 8.3.1 Free(¥0)
| 項目 | 値 |
|---|---|
| 売上 | ¥0 |
| OCR原価 | 5枚 × ¥0.8 = **¥4.0**(上限利用時) |
| 収支 | **−¥4.0 / 月**(上限まで使った場合) |

- Freeは**1ユーザー月最大¥4の赤字**。実際は5枚フル利用しないユーザーも多く、平均はさらに小さい。
- これは**獲得・転換のためのコスト**として許容範囲。広告なしでも軽微([1.10](./01-product-overview.md#110-収支シミュレーションの前提詳細は第8章)の「赤字にならない水準か」に対し、上限利用でも月¥4で収まる)。

### 8.3.2 Light(¥480/月)
| 項目 | 値 |
|---|---|
| 売上 | ¥480 |
| 決済手数料(30%) | −¥144 |
| 純売上 | ¥336 |
| OCR原価 | 30枚 × ¥0.8 = −¥24 |
| **粗利** | **¥312 / 月** |

- 粗利率 約65%。OCR単価が¥0.8でも十分黒字。OCRが¥0.3なら原価¥9でさらに改善。

### 8.3.3 Pro(¥780/月)
| 項目 | 100枚/月 | 150枚/月 |
|---|---|---|
| 売上 | ¥780 | ¥780 |
| 決済手数料(30%) | −¥234 | −¥234 |
| 純売上 | ¥546 | ¥546 |
| OCR原価(¥0.8/枚) | −¥80 | −¥120 |
| **粗利** | **¥466 / 月** | **¥426 / 月** |

- 150枚でも黒字。**「無制限」のリスクは極端なヘビーユーザー**(例:月500枚=原価¥400)。それでも純売上¥546に対し赤字にはならないが、利益は薄くなる。
- 対策:極端利用の監視(不正・業務転用の検知)、必要なら将来「公正利用ポリシー」を設ける。MVPは監視のみ。

### 8.3.4 決済手数料15%適用時(小規模事業者プログラム)
Apple/Google の小規模事業者プログラム(年間売上が一定以下なら手数料15%)が適用された場合、粗利は大きく改善する。OCR単価¥0.8・Pro 150枚での比較:

| 手数料率 | Light 粗利 | Pro 粗利(150枚) |
|---|---|---|
| 30%(基準) | ¥312 | ¥426 |
| 15%(小規模適用) | **¥384** | **¥543** |

- 個人開発の初期は売上規模が小さく**15%が適用される可能性が高い**。本章の基準試算(30%)は保守側であり、実際はこれより良くなる見込み。

## 8.4 OCR単価の感度分析(粗利への影響)

| OCR単価 | Light粗利 | Pro粗利(150枚) |
|---|---|---|
| ¥0.3 | ¥327 | ¥501 |
| ¥0.8 | ¥312 | ¥426 |
| ¥1.5 | ¥291 | ¥321 |
| ¥3.0 | ¥246 | ¥96 |

- **Light は OCR単価に鈍感**(枚数が少ない)。¥3でも黒字。
- **Pro は枚数が多く感度が高い**。OCR単価¥3・150枚で粗利¥96まで縮む。上位モデルを使う場合はProの粗利を要監視。
- 結論:**価格設定は OCR単価¥0.8 前提で十分成立**。コスト最適化(リサイズ・モデル選定・[6.8](./06-ocr-ai.md#68-コスト前提第8章へ接続))で余裕を作る。

## 8.5 全体収支イメージ(3ヶ月時点の仮シナリオ)

[第1章 1.9](./01-product-overview.md#19-成功指標初期仮説)の仮説(DL 5,000・転換率)に基づく粗い試算。

| 区分 | 人数(仮) | 月間粗利/人 | 月間粗利合計 |
|---|---|---|---|
| Free | 4,000 | −¥4 | −¥16,000 |
| Light | 300(Free→Light 転換) | ¥312 | ¥93,600 |
| Pro | 45(Light→Pro 15%) | ¥426 | ¥19,170 |
| **合計** | | | **約 ¥96,770 / 月** |

- ※DL 5,000 のうち課金到達は仮。**申告期(2〜3月)に転換が跳ねる**設計のため、年間では繁忙期に大きく寄る。
- この粗利から固定費(8.6)を引いた額が事業の手残り。

## 8.6 固定費・先行投資(月額イメージ)

| 項目 | 月額(仮) | 備考 |

| Supabase | ¥0〜数千 | 無料枠→Pro($25〜)へ。ユーザー増で増加 |
| エラー監視(Sentry等) | ¥0〜数千 | 無料枠あり |
| Apple Developer | 約¥1,100/月相当 | 年$99 |
| Google Play | 一括$25 | 初回のみ |
| ドメイン/その他 | 数百 | |

- MVP初期の固定費は**月数千円規模**に収まる見込み。8.5の粗利(月約¥9.7万・仮)で十分カバーできる。

## 8.7 損益分岐の目安

- 固定費を月¥10,000と仮定すると、**Light 約32人 または Pro 約24人**で固定費を回収(粗利ベース)。
- DL 5,000・Light転換300人の仮説が半分(150人)でも固定費は回収可能。**価格・原価構造は MVP として健全**。

## 8.8 リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| OCR単価の上振れ(上位モデル使用) | Pro粗利の縮小 | 画像リサイズ・モデル段階選定([6.3.3](./06-ocr-ai.md#633-モデル選定))・実測で見直し |
| 為替(円安) | API原価上昇 | 単価レンジで監視、必要なら価格改定 |
| 決済手数料 | 30%は重い | 小規模事業者プログラム(15%)の適用を確認 |
| Pro極端ヘビーユーザー | 利益希薄化 | 利用監視、将来の公正利用ポリシー |
| 転換率が仮説を下回る | 売上未達 | 申告期の訴求強化、オンボーディング改善 |
